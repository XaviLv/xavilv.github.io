    • Getting Your Matrix Dimensions Right

Takeaway: for a single input sample x, the dimension of the first layer’s weight matrix w is [n_w, n_x].

  • n_w : the number of hidden units in this layer
  • n_x : the dimension of the input feature vector And since all vectors in numpy are marked as the column vector, the w here should be considered as the one already transposed, like demonstrated in some paper.

All samples vectorized:

$$\begin{aligned} Z^{[1]} &= W^{[1]}A^{[0]} + b^{[1]} \newline A^{[1]} &= g^{[1]}(Z^{[1]}) \newline Z^{[2]} &= W^{[2]}A^{[1]} + b^{[2]} \newline A^{[2]} &= g^{[2]}(Z^{[2]}) \newline &… \newline \hat{Y} &= g^{[4]}(Z^{[4]}) = A^{[4]} \end{aligned} $$

where, $$ A^{[0]} = X $$


Getting Your Matrix Dimensions Right

As seen in the figure above, the annotation ‘n’ represents the number of units in each layer, including the input layer.

Let’s look at the first hidden layer.

$$ z^{[1]} = w^{[1]}x + b^{[1]} $$

So to make the matrix multiplication work, it’s easy to reach the following consequences.

  • Shape-of-z1: (n_1, 1)
  • Shape-of-w1: (n_1, n_x) or marked as (n_1, n_0)
  • Shape-of-x: (n_x, 1) or marked as (n_0, 1)
  • Shape-of-b1: (n_1, 1)

Similarly,

  • Shape-of-zL: (n_L, 1)
  • Shape-of-wL: (n_L, n_{L-1})
  • Shape-of-bL: (n_L, 1)

And if we vectorize the input samples by stacking them horizontally, then,

  • Shape-of-X: (n_x, m)
  • Shape-of-ZL: (n_L, m)
  • Shape-of-WL: (n_L, n_{L-1})
  • Shape-of-BL: (n_L, m)

So, by checking the dimensions of the parameters, we can find out the potential mistakes in the early stage.
